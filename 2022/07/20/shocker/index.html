HTB: Shocker

k1ckstand
2022-07-20
HTB
2014-6271, cve, linux

Shocker is an easy box that requires you to discover a shell script in the cgi-bin directory and pice together that the box is vulnerable to shellshock. Getting root requires you to find that the user www-data can run perl as root. Let’s get started.

Getting User

Start out by adding the box to our host file.

Recon

Taking a quick look at the site we see there is not much going on.

Let’s take a closer look at the cgi-bin directory. Because the cgi-bin directory most likely to contain files we need to set our search to include a few extensions.

Here we find a user.sh script.

Given the box title (and age) combined with a shell script in cgi-bin and the subtle hint on the main page we can try to give shellshock a spin. Make sure to include the full path to any commands used.

Metasploit

Given the popularity of shellshock there is a metasploit module that seems to work well.
just search for shellshock and set the required options.

There is also an nmap script http-shellshock.nse that does work but I found it a bit difficult to use. It involves setting up a proxy to intercept request where the script adds the exploit to the header.
